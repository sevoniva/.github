# Sevoniva Roadmap

This roadmap lists public project directions. It is not a delivery contract.

## Now

- Stabilize the GitHub organization portal and public project catalog.
- Keep core repository READMEs aligned with their current scope.
- Improve OceanBase MCP documentation, examples, and release notes.
- Keep Crest public status and release notes accurate.
- Maintain reusable Codex skills for requirement review and test case generation.

## Next

- Add clearer quick-start paths for Crest, oceanbase-mcp, skills, cogniflow, Koravo, and Nivora.
- Improve issue templates and contribution paths based on real feedback.
- Add lightweight project status sections to public repositories.
- Document common MCP, BI, workflow, and DevOps use cases.

## Later

- Publish more examples for MCP integrations and Codex skills.
- Add public demos or screenshots where they help explain product scope.
- Split mature docs into the documentation site when repository READMEs become too long.
- Consider GitHub Discussions for projects with active external users.
