# Code of Conduct

Sevoniva projects are maintained as practical engineering workspaces. Keep discussions focused, specific, and respectful.

## Expected behavior

- Be clear about the repository, version, environment, and use case you are discussing.
- Challenge ideas and implementation details, not people.
- Keep issue and pull request comments actionable.
- Respect maintainers' time and project scope.
- Do not publish secrets, credentials, private data, or vulnerability details in public issues.

## Unacceptable behavior

- Harassment, insults, threats, or discriminatory language.
- Spam, off-topic promotion, or repeated low-effort comments.
- Sharing private information without permission.
- Public disclosure of security vulnerabilities before maintainers have had time to respond.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues; lock discussions; or block accounts when behavior disrupts project work or creates risk for users.

## 中文

Sevoniva 项目是面向实际工程工作的协作空间。讨论应保持具体、清晰、尊重事实。

我们期望：

- 说明相关仓库、版本、环境和使用场景。
- 讨论方案和实现，不攻击个人。
- issue 和 pull request 评论保持可执行。
- 尊重维护者时间和项目边界。
- 不在公开 issue 中发布密钥、凭证、私有数据或漏洞细节。

不可接受：

- 骚扰、辱骂、威胁或歧视性表达。
- 垃圾信息、无关推广或重复低质量评论。
- 未经允许公开他人隐私信息。
- 在维护者响应前公开披露安全漏洞细节。

维护者可以编辑、隐藏或删除评论，关闭 issue，锁定讨论，或阻止影响项目协作和用户安全的账号。
