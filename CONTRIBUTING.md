# Contributing

Thanks for taking the time to improve a Sevoniva project.

## Before opening an issue

- Check whether the issue belongs to a specific repository.
- Search existing issues before opening a new one.
- Include the project version, runtime, database, operating system, and logs when they matter.
- For security issues, do not open a public issue. See [SECURITY.md](SECURITY.md).

## Pull requests

- Keep changes focused on one problem.
- Include tests or a clear manual verification note when behavior changes.
- Update documentation when commands, configuration, APIs, or user-facing behavior changes.
- Avoid broad refactors in feature or bug-fix pull requests.

## 中文

感谢你改进 Sevoniva 项目。

提交 issue 前：

- 先确认问题属于哪个具体仓库。
- 先搜索已有 issue，避免重复提交。
- 涉及环境问题时，请提供项目版本、运行环境、数据库、操作系统和相关日志。
- 安全问题不要提交公开 issue，请查看 [SECURITY.md](SECURITY.md)。

提交 pull request 时：

- 一个 PR 只解决一个清晰问题。
- 行为有变化时，请附带测试或明确的手动验证说明。
- 命令、配置、API 或用户可见行为变化时，请同步更新文档。
- 避免在功能或修复 PR 里混入大范围重构。
